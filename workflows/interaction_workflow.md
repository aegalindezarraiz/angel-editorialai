# Interaction Workflow

## Objective
Define how the user, orchestrator agent, specialist agents, providers, MCP tools, and publishing systems interact across one full content cycle.

## Primary Actors
- User
- Orchestrator Agent
- Trend Intelligence Agent
- Market Research Agent
- Audience Insight Agent
- Editorial Strategy Agent
- Blog Agent
- LinkedIn Agent
- Instagram Agent
- TikTok Agent
- Creative QA Agent
- Publishing Agent
- Performance Analyst Agent
- Memory Agent
- External providers and MCP tools

## Workflow A: Strategic Content Mission

### Step 1: Mission Intake
User provides:
- topic or objective
- audience
- offer or brand focus
- desired channels
- campaign goal
- deadline or publishing cadence

System returns:
- mission summary
- detected channels
- required assets
- recommended workflow path

### Step 2: Intelligence Stage
Orchestrator triggers:
- Trend Intelligence Agent
- Market Research Agent
- Audience Insight Agent

Outputs:
- trend opportunities
- content angles
- audience language
- objections
- proof points

### Step 3: Editorial Strategy
Editorial Strategy Agent converts intelligence into:
- thesis
- promise
- content pillar
- CTA direction
- repurposing map

### Step 4: Source Asset Creation
Blog Agent creates the source article.
This source asset becomes the canonical content unit.

### Step 5: Channel Derivation
Orchestrator dispatches source asset to:
- LinkedIn Agent
- Instagram Agent
- TikTok Agent

Outputs:
- LinkedIn post set
- Instagram carousel, caption, reel brief
- TikTok script set
- Blog article package

### Step 6: Quality Gate
Creative QA Agent validates all assets.

If failed:
- Orchestrator routes revision tasks to the responsible agent.

If approved:
- assets move to packaging.

### Step 7: Packaging and Publishing
Publishing Agent prepares:
- final copy
- asset checklist
- schedule notes
- platform-ready payloads

### Step 8: Measurement and Learning
Performance Analyst Agent collects metrics.
Memory Agent stores:
- hooks
- CTAs
- winning topics
- winning formats

## Workflow B: Fast Trend Response
1. User selects trend mode.
2. Trend Intelligence Agent checks urgency and relevance.
3. Editorial Strategy Agent compresses thesis.
4. LinkedIn, Instagram, and TikTok Agents create short-cycle assets.
5. QA validates.
6. Publishing Agent schedules same-day release.

## Workflow C: Content Repurposing
1. User uploads source material or selects past content.
2. System extracts canonical insight.
3. Channel agents rebuild assets by platform.
4. QA and packaging complete the output.

## Workflow D: Performance Optimization Loop
1. User opens analytics mode.
2. Performance Analyst Agent identifies winners and failures.
3. Memory Agent updates strategic memory.
4. Orchestrator proposes next-cycle themes, hooks, and CTA variations.

## Human Control Points
The user can intervene at:
- mission approval
- thesis approval
- source asset approval
- channel asset approval
- publishing approval
- analytics review

## Interaction Principle
The system should default to guided autonomy:
- autonomous in analysis and drafting
- supervised at decision checkpoints
- transparent in agent outputs and revision paths
